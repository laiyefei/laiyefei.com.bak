```java
public class HelloWorld{
  public static void main(String[] args){//参数还可以是：String args[]
    System.out.println("Hello World");
  }
}
```

~~~
运行结果：
> javac HelloWorld.java
> java HelloWorld
HelloWorld
~~~


## 定义

无论如何，还是先抛个定义：

~~~
语法是语言学的一个分支，研究按确定用法来运用的"词类"、"词"的屈折变化或表示相互关系的其他手段以及词在句中的功能和关系。
包含词的构词、构形的规则和组词成句的规则。语法有两个含义，一指语法结构规律本身，即语法事实；一指语法学，是探索并描写
语法结构的科学，是语法学者对客观存在的语法体系的认识和说明。语法事实本身没有分歧，但由于语法学者占有的材料，观察角度，
分析方法不一致，语法学体系是有分歧的。语法包括词法和句法两部分。词法主要是指词的构成，变化和分类规律。句法主要是指短语
和句子等语法单位的构成和变化规则。
~~~

## 解析


### 概念

· 一些基本概念
~~~
类：类是一个模板，用来描述某一具体事物的通性。
对象：对象是一个类的实例，有状态和行为。【例：一条具体的狗是一个对象，他的状态有（颜色，名字，种类，...）；行为有（摇尾，叫，吃，...）】
属性：属性是状态，是特征，是某一个点的描述。
方法：方法就是行为。是某一些具体操作的动作。
实例变量：每个对象都有独特的实例变量，对象的状态由这些实例变量的值决定的。
~~~

· 一些注意点
~~~

基本要点
-------------------------------------------------------------------------------------------------------------------------------------------
- 大小写敏感：Java是大小写敏感的【如，hello和Hello不一样】。
- 类名：首字母大写【非强制性，只是编码习惯】。
- 方法名：所有方法以小写字母开头【非强制性，只是编码习惯】。
- 源文件名：源文件名必须与类名相同，不相同会报错【注意大小写敏感】。
- 入口函数：所有Java程序由 public static void main(String[] args) 方法开始执行。
==========================================================================================================================================

标识符注意点
-------------------------------------------------------------------------------------------------------------------------------------------
- 所有标识符都应该以字母【A-Z或者a-z】，美元符【$】，或者下划线【_】开始。
- 首字母之后可以是字母【A-Z或者a-z】，美元符【$】，或者下划线【_】或者数字的任何字符组合。
- 关键字不能用作标识符。
- 标识符是大小写敏感的。
- 合法标识符举例：sherlock，$sherlock，_sherlock，_221_sherlock，...
- 非法标识符举例：221sherlock，-sherlock，...
==========================================================================================================================================

修饰符要点
------------------------------------------------------------------------------------------------------------------------------------------
- 访问控制修饰符：default，public，protected，private
     > 权限列表

          修 饰 符  |  当前类  |   同一包内   |   子孙类   |   其他包
        ———————————+——————————+—————————————+————————————+—————————————————————————...
          public   |    Y     |     Y       |     Y      |    Y
        ———————————+——————————+—————————————+————————————+—————————————————————————...
         protected |    Y     |     Y       |     Y      |    N
        ———————————+——————————+—————————————+————————————+—————————————————————————...
          default  |    Y     |     Y       |     N      |    N
        ———————————+——————————+—————————————+————————————+—————————————————————————...
          private  |    Y     |     N       |     N      |    N
        ———————————————————————————————————————————————————————————————————————————...

- 非访问控制修饰符：final，abstract，strictfp
     > static 修饰符，用来创建类方法和类变量
     > final 修饰符，用来修饰类、方法和变量，final 修饰的类不能够被继承，修饰的方法不能被继承类重新定义，修饰的变量为常量，是不可修改的
     > abstract 修饰符，用来创建抽象类和抽象方法
     > synchronized 和 volatile 修饰符，主要用于线程的编程
==========================================================================================================================================

变量
------------------------------------------------------------------------------------------------------------------------------------------
- 局部变量
     > 局部变量声明在方法、构造方法或者语句块中
     > 局部变量在方法、构造方法、或者语句块被执行的时候创建，当它们执行完成后，变量将会被销毁
     > 访问修饰符不能用于局部变量
     > 局部变量只在声明它的方法、构造方法或者语句块中可见
     > 局部变量是在栈上分配的
     > 局部变量如果没有默认值，则局部变量被声明后，必须经过初始化，才可以使用
- 类变量【静态变量】
     > 类变量也称为静态变量，在类中以static关键字声明，但必须在方法构造方法和语句块之外
     > 无论一个类创建了多少个对象，类只拥有类变量的一份拷贝
     > 静态变量除了被声明为常量外很少使用。【常量是指声明为public/private，final和static类型的变量。常量初始化后不可改变】
     > 静态变量储存在静态存储区。【经常被声明为常量，很少单独使用static声明变量】
     > 静态变量在程序开始时创建，在程序结束时销毁
     > 与实例变量具有相似的可见性。但为了对类的使用者可见，大多数静态变量声明为public类型
     > 默认值和实例变量相似。【数值型变量默认值是0，布尔型默认值是false，引用类型默认值是null。变量的值可以在声明的时候指定，也可以在构造方法中指定。此外，静态变量还可以在静态语句块中初始化】
     > 静态变量可以通过：ClassName.VariableName 的方式访问
     > 类变量被声明为public static final类型时，类变量名称一般建议使用大写字母。如果静态变量不是public和final类型，其命名方式与实例变量以及局部变量的命名方式一致
- 成员变量【非静态变量，实例变量】
     > 实例变量声明在一个类中，但在方法、构造方法和语句块之外
     > 当一个对象被实例化之后，每个实例变量的值就跟着确定
     > 实例变量在对象创建的时候创建，在对象被销毁的时候销毁
     > 实例变量的值应该至少被一个方法、构造方法或者语句块引用，使得外部能够通过这些方式获取实例变量信息
     > 实例变量可以声明在使用前或者使用后
     > 访问修饰符可以修饰实例变量
     > 实例变量对于类中的方法、构造方法或者语句块是可见的。一般情况下应该把实例变量设为私有。通过使用访问修饰符可以使实例变量对子类可见
     > 实例变量具有默认值。数值型变量的默认值是0，布尔型变量的默认值是false，引用类型变量的默认值是null。变量的值可以在声明时指定，也可以在构造方法中指定
     > 实例变量可以直接通过变量名访问。但在静态方法以及其他类中，就应该使用完全限定名：ObejectReference.VariableName

ps.
     > 静态变量属于类，该类不生产对象，通过类名就可以调用静态变量
     > 实例变量属于该类的对象，必须产生该类对象，才能调用实例变量
     > 实例变量属于某个对象的属性，必须创建了实例对象，其中的实例变量才会被分配空间，才能使用这个实例变量
     > 静态变量不属于某个实例对象，而是属于类，所以也称为类变量，只要程序加载了类的字节码，不用创建任何实例对象，静态变量就会被分配空间，静态变量就可以被使用了
     > 静态变量是用来声明规则的，一旦固定都是用用来引用的【类似社会中的法律，规定好后你只能拿来说。但是也可以改的，通过重新的声明法律就行。】。
     > 实例变量在整个类内部是可访问的，而不管实例变量声明在类的哪个位置
     > 无final修饰，声明时赋值，构造器中赋值，静态语句块 或 静态方法赋值
     > 有final修饰，声明时赋值。【声明与赋值分开可在静态语句块中赋值】

### 数据类型和数据结构

* 基本数据类型

~~~

- 内置数据类型
     1. byte：
        最小值：-128【-2^7】
        最大值：127【2^7 - 1】
        默认值：0
        用途：用在大型数组中节约空间，主要代替整数。【因为 byte 只占用int的1/4】【例：byte a = 100，byte b = -50。】
     2. short
        最小值：-32768【-2^15】
        最大值：32767【2^15 - 1】
        默认值：0
        用途：用在大型数组中节约空间，主要代替整数。【因为 byte 只占用int的1/2】【例：short s = 1000，short r = -20000。】
     3. int
        最小值：-2147483648【-2^31】
        最大值：2147483647【2^31 - 1】
        默认值：0
        用途：一般整型变量默认为int型，这可能和系统有关系。【例：int a = 100000, int b = -200000。】
     4. long
        最小值：-9223372036854775808【-2^63】
        最大值：9223372036854775807【2^63 - 1】
        默认值：0L
        用途：这种类型主要使用在需要比较大整数的系统上。【例：long a = 100000L，Long b = -200000L。】【ps."L"理论上不分大小写，但是若写成"l"容易与数字"1"混淆，不容易分辩。所以最好大写。】
     5. float
        特征：单精度、32位、符合IEEE 754标准的浮点数
        默认值：0.0f
        用途：大型浮点数组的时候可节省内存空间。【例：float f1 = 234.5f。】【ps. 浮点数不能用来表示精确的值，如货币；】
     6. double
        特征：双精度、64 位、符合IEEE 754标准的浮点数
        默认值：0.0d
        用途：浮点数的默认类型为double类型。【例：double d1 = 123.4】【ps. 浮点数不能用来表示精确的值，如货币；】

- 自动类型转换

    低     ------------>>>>>>>>>>>>>------------高级
————————————————————————————————————————————————————————————————...
    byte , short , char → int → long → float → double
————————————————————————————————————————————————————————————————...
    规则记录：
        1. 不能对boolean类型进行类型转换。
        2. 不能把对象类型转换成不相关类的对象。
        3. 在把容量大的类型转换为容量小的类型时必须使用强制类型转换。
        4. 必须满足转换前的数据类型的位数要低于转换后的数据类型【例：short数据类型的位数为16位，就可以自动转换位数为32的int类型，同样float数据类型的位数为32，可以自动转换为64位的double类型。】。

- 强制类型转换
    规则记录：
        1. 条件是转换的数据类型必须是兼容的。
        2. 转换过程中可能导致溢出或损失精度。【例：int i =128; byte b = (byte)i;】
        3. 浮点数到整数的转换是通过舍弃小数得到，而不是四舍五入【例：(int)23.7 == 23; (int)-45.89f == -45】        
        4. (type)value type是要强制类型转换后的数据类型。

- 隐含强制类型转换
        1. 整数类型的默认值都是 int
        2. 浮点型不存在这种情况，因为在定义 float 类型时必须在数字后面跟上 F 或者 f。

* 数据结构

~~~

枚举【Enumeration】
------------------------------------------------------------------------------------------------------------------------------------------
- 概念：限制变量只能是预先设定好的值。
- 作用：使用枚举可以减少代码中的bug。

注：枚举可以单独声明或者声明在类里面。方法、变量、构造函数也可以在枚举中定义。
==========================================================================================================================================

位集合【BitSet】
------------------------------------------------------------------------------------------------------------------------------------------
- 概念：位集合类实现了一组可以单独设置和清除的位或标志。
- 作用：该类在处理一组布尔值的时候非常有用，你只需要给每个值赋值一"位"，然后对位进行适当的设置或清除，就可以对布尔值进行操作了。
==========================================================================================================================================

向量【Vector】
------------------------------------------------------------------------------------------------------------------------------------------
- 概念：向量（Vector）类和传统数组非常相似，但是Vector的大小能根据需要动态的变化。
- 作用：是在创建对象的时候不必给对象指定大小，它的大小会根据需要动态的变化
==========================================================================================================================================

栈【Stack】
------------------------------------------------------------------------------------------------------------------------------------------
- 概念：实现了一个后进先出（LIFO）的数据结构
- 作用：倒序等
==========================================================================================================================================

字典【Dictionary】
------------------------------------------------------------------------------------------------------------------------------------------
- 概念：键映射到值的数据结构
- 作用：通过特定的键而不是整数索引来访问数据的时候
==========================================================================================================================================

哈希表【Hashtable】
------------------------------------------------------------------------------------------------------------------------------------------
- 概念：用户定义键结构的基础上来组织数据的手段
- 作用：在地址列表的哈希表中，你可以根据邮政编码作为键来存储和排序数据，而不是通过人名。
==========================================================================================================================================

属性【Properties】
------------------------------------------------------------------------------------------------------------------------------------------
- 概念：继承于 Hashtable.Properties 类表示了一个持久的属性集.属性列表中每个键及其对应值都是一个字符串
- 作用：在获取环境变量时它就作为System.getProperties()方法的返回值
==========================================================================================================================================

### 语法

前面简单介绍和罗列了Java的一些概念，接下来才是重点【逼格的体现，就看语法怎么写了 ← _ ← 】。

#### 循环结构

* while 循环

```java
while([布尔表达式]){
  //循环体
}
```

* do ... while 循环

```java
do {
  //循环体
}while([布尔表达式])
```

* for 循环

```java
// 第一种
for([表达式1]; [布尔表达式]; [表达式2]){
  //循环体
}
// 第二种
for([声明语句]:[表达式【返回数组】]){
 //循环体
}
```

#### 分支结构

* if 语句

```java
if([布尔表达式]){
  //语句块
}
```

* if ... else ... 语句

```java
if([布尔表达式]){
  //如果true的语句块
}else{
  //false的语句块
}
```

* if ... else if ... else ... 语句

```java
if([布尔表达式1]){
  //布尔表达式1为true执行的语句块
}else if([布尔表达式2]){
  //布尔表达式2为true执行的语句块
}else{
  //else执行的语句块
}
```

* 嵌套 if ... else ... 语句

```java
if ([布尔表达式]){
  //TODO..
  if([布尔表达式]){
    //TODO..
  }else{
    //TODO...
  }
}else{
  //TODO..
}
```

* switch 语句

```java
switch([表达式]){
  case [值1]:
       //语句
       break;//可选
  case [值2]:
       //语句
       break;//可选
  //...
  default:
      //默认执行语句
}
```
